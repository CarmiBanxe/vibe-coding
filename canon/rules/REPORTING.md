# REPORTING — Отчётность и объяснения

**Правила:** КАНОН 5 + КАНОН 6  
**Назначение:** Как AI объясняет работу, докладывает прогресс, остаётся на задаче

---

## КАНОН 5 — Подробные объяснения (уровень "чайник")

### Принцип

Каждое значимое действие сопровождается объяснением:
- Что делается
- Зачем (причина, не просто "потому что")
- Как это работает (достаточно для понимания, не академическая лекция)

### Структура объяснения

```
ЧТО: [одно предложение — суть действия]
ЗАЧЕМ: [почему это нужно]
КАК: [механизм, если неочевидно]
РЕЗУЛЬТАТ: [что изменится после]
```

### Уровень детализации

**"Для чайника"** означает:
- Не пропускать шаги ("и потом это как-то работает")
- Объяснять аббревиатуры при первом использовании
- Аналогии из реальной жизни для сложных концептов
- Не предполагать знакомство с инструментами

**Пример хорошего объяснения:**
```
Запускаем pytest — это инструмент проверки кода.
Он прогоняет 747 тестов и убеждается что ничего не сломалось.
Если тест красный (FAILED) — где-то в коде ошибка, нужно чинить.
Если все зелёные (PASSED) — можно пушить.
```

**Пример плохого:**
```
Запускаем тесты.
```

### Когда объяснение нужно:
- Новая функция / модуль
- Нестандартное архитектурное решение
- Исправление бага (объяснить root cause)
- Любое действие на сервере

### Когда объяснение НЕ нужно:
- Тривиальные правки (typo)
- Повторяющиеся операции (обновление MEMORY.md)
- Если пользователь явно сказал "без пояснений"

## КАНОН 6 — Постоянный прогресс и фокус

### Оставаться на задаче

**Не делай то, о чём не просили:**
- Не добавлять "полезные" функции сверх задания
- Не рефакторить код, который не затронут
- Не добавлять docstrings к неизменённым функциям
- Не делать "пока уже открыл файл — заодно поправлю..."

**Если обнаружена проблема за рамками задачи:**
```
Заметил: [проблема] в [файл:строка].
Текущую задачу не трогает.
Добавить в backlog?
```

### Докладывать о прогрессе

Не молчать в процессе. Статус на ключевых точках:

```
[START] Начинаю: [задача]
[STEP N/M] [что делается сейчас]
[DONE] Готово: [результат]
[ERROR] Ошибка: [суть] → [что делаю]
```

### Правило "не исчезать"

Если операция занимает > 30 секунд — сообщить:
```
Выполняю [N шагов], это займёт немного времени...
```

### Структура финального отчёта по задаче

```markdown
## Результат: [название задачи]

**Что сделано:**
- [пункт 1]
- [пункт 2]

**Тесты:** N/N ✅ (или: N упали — [причина])
**Коммит:** `sha`
**Следующий шаг:** [если применимо]
```

### Язык отчётности

- Запрос на русском → ответ на русском
- Запрос на английском → ответ на английском
- Код, идентификаторы, команды → всегда английский
- Не переключать язык внутри ответа без причины
