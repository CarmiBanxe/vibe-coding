# COLLABORATION — Коллаборация агентов

**Правила:** КАНОН 1 + КАНОН 9  
**Назначение:** Роли агентов, разделение обязанностей, изоляция проектов

---

## КАНОН 1 — Роли агентов

### Иерархия агентов

```
Пользователь (CEO Moriel Carmi)
    → финальное слово по всем решениям
    → видит только результат, не процесс координации

Claude Code (Архитектор / Оркестратор / Ревьюер)
    → проектирует решения
    → координирует Aider
    → проверяет качество
    → коммитит и пушит

Aider (Исполнитель)
    → пишет код по заданию Claude Code
    → вызывается через MCP автоматически
    → не взаимодействует с пользователем напрямую
```

### Принцип "пользователь не координирует агентов"

Пользователь НЕ должен:
- Вручную вызывать Aider
- Указывать какой агент что делает
- Координировать Claude Code ↔ Aider переключение
- Видеть внутренние протоколы коллаборации

Пользователь видит ТОЛЬКО:
- Финальный результат
- Статус задачи (выполнено / ошибка / требуется подтверждение)
- Краткое объяснение что было сделано

### Когда Claude Code выполняет сам vs делегирует Aider

**Claude Code выполняет сам:**
- Архитектурные решения
- Ревью кода
- Git операции
- MEMORY.md обновление
- Простые правки (< 20 строк, 1 файл)

**Делегирует Aider:**
- Написание нового кода (> 50 строк)
- Рефакторинг больших файлов
- Генерация тестов по шаблону
- Массовые правки по нескольким файлам

## КАНОН 9 — Один терминал = один проект

### Правило изоляции

```
Один терминал → один проект → один репозиторий
```

Несколько терминалов = РАЗНЫЕ проекты в РАЗНЫХ репозиториях.

### Что означает изоляция:

1. **Не читать** файлы чужого проекта
2. **Не цитировать** решения из другого проекта
3. **Не заимствовать** контекст, архитектуру, паттерны без явной команды
4. **Не переносить** файлы между проектами без явного указания обоих проектов
5. **Не смешивать** CLAUDE.md инструкции разных проектов

### Проекты и их изоляция

| Проект | Репозиторий | Статус |
|--------|-------------|--------|
| BANXE (vibe-coding) | CarmiBanxe/vibe-coding | PRIMARY (этот терминал) |
| banxe-architecture | CarmiBanxe/banxe-architecture | SUB (только для arch docs) |
| developer-core | CarmiBanxe/developer-core | CANON + tools |
| GUIYON | (отдельно) | **АБСОЛЮТНЫЙ ЗАПРЕТ** — чужой проект |

### GUIYON — особый статус

GUIYON — абсолютно чужой проект. Запрет установлен CEO 2026-04-01.

- Не читать файлы GUIYON
- Не писать в GUIYON
- Не обсуждать GUIYON
- Не создавать файлы для GUIYON
- `mcp__guiyon-files__*` инструменты — **ЗАПРЕЩЕНЫ**

Исключение: SSH-команды через `ssh gmktec "..."` к `/data/guiyon-project/` — разрешены
(только если пользователь явно просит и это связано с инфраструктурой, не кодом проекта).

### Явная команда на перенос

Единственный способ легально перенести контент между проектами:

```
Пользователь явно указывает:
"Скопируй [файл/функцию] из [проект A] в [проект B]"

Тогда допустимо:
rsync -av ~/projectA/path/ ~/projectB/path/
```

Без такой команды — перенос запрещён.
