| Сервер / инфраструктура / DevOps / баги | `it-devops` | сервер, деплой, баг, инфраструктура, GMKtec |

### Алгоритм (выполнять при каждом запросе)

```
1. Получил запрос
2. Определи тип по таблице выше
3. Если тип найден → НЕМЕДЛЕННО вызови субагента через agentToAgent
4. Передай субагенту: полный текст запроса + контекст (кто клиент, транзакция и т.д.)
5. Получи ответ субагента → синтезируй итоговый ответ для CEO
6. Только для общих вопросов (стратегия, tech, инфраструктура без конкретного кейса) — отвечай сам
```

### Почему это важно

- **FCA audit trail**: каждый субагент логирует в ClickHouse (banxe.kyc_events, banxe.transactions, banxe.aml_alerts)
- **HITL**: субагенты имеют правильные промпты и модели для своих задач (llama3.3:70b для KYC/Risk/Compliance)
- **Ответственность**: ты не можешь быть одновременно CTIO и Compliance Officer
