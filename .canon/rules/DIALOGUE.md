# DIALOGUE — Протоколы диалога

**Правила:** КАНОН 2 + КАНОН 7  
**Назначение:** Как AI ведёт диалог с пользователем, задаёт вопросы, предлагает решения

---

## КАНОН 2 — Протокол Question→Explain→Confirm→Act

### Применяется для:
- Необратимых действий (удаление, деплой, DROP TABLE, force push)
- Действий на удалённых серверах (SSH: GMKtec, NucBox, любой сервер)
- Изменения критичных конфигов (SOUL.md, .env, openclaw.json)
- Любого действия, которое пользователь явно не назвал

### Последовательность:

```
1. ВОПРОС (если задача неоднозначна):
   "Уточни: [конкретный вопрос]?"
   Максимум 1-2 вопроса за раз — не допрос.

2. ОБЪЯСНЕНИЕ:
   "Собираюсь сделать: [N шагов]"
   "Почему: [1-2 причины]"
   "Риски: [если есть]"

3. ПОДТВЕРЖДЕНИЕ:
   "Подтверди: да / нет"
   → СТОП. Жду явного ответа.

4. ДЕЙСТВИЕ:
   После "да" / "ок" / "давай" / "go" — выполняю.

5. РЕЗУЛЬТАТ:
   "Готово: [что сделано]"
   "Проверка: [тест / статус]"
```

### Триггеры ДЛЯ QRAA (обязательно ждать подтверждения):
- SSH-команды на сервер
- `systemctl restart`, `docker restart`
- Изменение `*.yaml`, `.env`, `SOUL.md`, `openclaw.json`
- `rm`, `DROP`, `TRUNCATE`, force push
- Деплой на production

### Без QRAA (выполнять сразу):
- `ls`, `cat`, `grep`, `find`, `curl GET`, `git status`, `git log`
- Правки кода в текущем репозитории
- `git add`, `git commit`, `git push` (текущий репо)
- `pytest`
- Создание новых файлов (не конфигурационных)

## КАНОН 7 — Консультационный протокол

### Применяется для:
- Юридических вопросов (LEGAL-профиль)
- Архитектурных решений (CLASS_D)
- Стратегических выборов (несколько равнозначных вариантов)
- Любых вопросов, где пользователь должен принять решение сам

### Протокол:

```
ПРЕДЛОЖЕНИЕ: [что рекомендую]
ОБОСНОВАНИЕ: [почему именно это]
АЛЬТЕРНАТИВЫ: [если есть]
ЖДУ РЕШЕНИЯ: [от пользователя]
```

### Правила:
- AI предлагает — пользователь одобряет
- Не реализовывать стратегическое решение без явного "да"
- Если пользователь не отвечает на вопрос — напомнить один раз, потом действовать по лучшему варианту

### Запрещено в диалоге:
| Нельзя | Правильно |
|--------|-----------|
| "Хочешь запушить?" | Пуши сам |
| "Выбери вариант A/B/C" | Реализуй все нужные шаги сам |
| "Что делать дальше?" | Доведи задачу до конца |
| Предлагать ручные git-команды | Выполняй сам |
| Задавать >2 уточняющих вопросов | Принять лучшее решение самостоятельно |

---

## Правило эскалации

Если после 3 попыток задача не решается — эскалировать пользователю:

```
СТОП — требуется решение.
Проблема: [суть]
Что уже пробовал: [список]
Нужно от тебя: [конкретное действие или разрешение]
```

Остановиться ≠ провал. Остановиться вовремя = профессионализм.
